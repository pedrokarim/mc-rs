Overview

MC-RS is a from-scratch Minecraft Bedrock Edition server written entirely in Rust, targeting full survival gameplay with high performance and memory safety.

What is MC-RS?

MC-RS is a complete Minecraft Bedrock Edition server implementation written in Rust. It targets protocol version 766 (Bedrock Edition 1.21.50) and implements the full survival gameplay experience — not just a proxy or relay.

The server handles everything from the low-level RakNet UDP transport and ECDH encryption handshake through to world generation, mob AI, combat, crafting, enchantments, and a full plugin system. Players can connect with an unmodified Bedrock client (Windows 10, mobile, or console) and experience survival mode with:

  • Procedurally generated terrain with 10 biomes, caves, ores, and structures
  • Full combat system with armor, enchantments, critical hits, and potion effects
  • Crafting tables, furnaces, anvils, enchanting tables, stonecutters, grindstones, and looms
  • Mob AI with pathfinding, breeding, and natural spawn/despawn cycles
  • Three dimensions: Overworld, Nether, and The End with portal mechanics
  • Plugin support via Rust API, WASM (wasmtime), and Lua (mlua)
Note: MC-RS is a clean-room implementation. It does not use or depend on Mojang's official server code (BDS). All protocol handling is implemented from public documentation and packet analysis.

Why Rust?

Performance

Compiled to native code with zero-cost abstractions. No garbage collector pauses, no JIT warmup. Chunk generation and packet processing run at full speed from the first tick.

Memory Safety

Rust's ownership system eliminates use-after-free, double-free, buffer overflows, and data races at compile time. No null pointer exceptions, no segfaults in production.

Fearless Concurrency

Send and Sync traits prevent data races. Combined with tokio's async runtime, MC-RS handles hundreds of concurrent connections without thread-safety bugs.

Type System

Rust's expressive type system catches protocol encoding bugs, invalid state transitions, and missing packet fields at compile time rather than in production.

MC-RS has been developed in 5 major phases, each building on the previous:

PhaseScopeHighlights
Phase 1 Foundation RakNet transport, protocol codec, login flow, encryption, world init, chunks, movement, blocks, inventory, chat, commands, dynamic chunk loading
Phase 2 Multiplayer Multi-player sync, PvP combat, 40+ commands with entity selectors, permissions system (whitelist, ban, ops)
Phase 3 Gameplay ECS entities, mob AI (9 behaviors), full combat (armor, enchantments, potions), survival (hunger, drowning, fall damage), crafting (50+ recipes), world generation (10 biomes, caves, ores, structures, Nether, End), persistence (LevelDB), block ticks (fluids, gravity, redstone, pistons), XP, weather, furnaces, enchanting tables, anvils
Phase 4 Extensibility Rust plugin API (15 events), WASM runtime (wasmtime), Lua scripting (mlua), behavior pack loader, forms UI, hot-reload
Phase 5 Operations Advanced commands (/setblock, /fill, /clone, /execute, /scoreboard), RCON, Query protocol, console REPL, ticking areas, multi-dimension portals, anti-cheat, performance optimizations
